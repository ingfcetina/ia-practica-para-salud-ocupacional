# Guion del taller de una hora

tags: #guion #clase #taller

Agenda sugerida para una clase de 60 minutos.

| Tiempo | Bloque | Objetivo |
|---:|---|---|
| 0-5 min | Apertura | Bajar ansiedad técnica y fijar reglas de privacidad. |
| 5-15 min | IA como copiloto | Mostrar ChatGPT/Claude con un problema profesional. |
| 15-30 min | Python mínimo | Leer un CSV, agrupar valores y graficar. |
| 30-40 min | Codex | Pasar de una idea a código revisable. |
| 40-50 min | Web interactiva | Mostrar checklist en HTML. |
| 50-60 min | Cierre | Método, límites y próximos pasos. |

## Mensaje de apertura

> Hoy no vamos a aprender toda la programación. Vamos a aprender a pedir ayuda de manera precisa para crear soluciones pequeñas, útiles y seguras.

## Demo recomendada

1. Abrir [[02-practicas-guiadas/practica-01-analizar-ausentismo|Práctica 1: analizar ausentismo]].
2. Copiar un prompt de [[04-herramientas-ia/chatgpt-web|ChatGPT web]].
3. Revisar [[03-python-desde-cero/leer-datos-csv|Leer datos CSV]].
4. Abrir `assets/checklist_interactivo.html`.

## Nota para facilitar la clase

Para principiantes, no conviene gastar la hora instalando herramientas. Antes de la sesión:

- ten Python, `pandas` y `matplotlib` instalados en el equipo de demostración;
- prepara una captura o salida esperada por si falla la ejecución en vivo;
- presenta la ejecución de Python como demostración opcional si el grupo no tiene ambiente listo;
- reserva el tiempo del grupo para entender el método, los riesgos y los ejemplos aplicables.

## Secuencia mental

```text
Problema profesional → datos seguros → prompt claro → prototipo → validación humana → mejora
```
